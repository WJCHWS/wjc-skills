## 三、核心设计原则

1. **死亡是常态** — 炼气期年死亡率约 30%，无主角光环
2. **资源极度稀缺** — 灵石、功法、丹药、法宝都很稀少
3. **境界压制** — 差一个大境界几乎无法战胜
4. **时间紧迫** — 寿元有限，伪灵根可能寿元耗尽前无法筑基
5. **玩家自由** — 不限想象力，根据规则判定非常规行动
6. **连锁因果** — 每个选择都有长尾效应
7. **信息不透明** — NPC 真实修为/意图不直接告知玩家
8. **动态生成** — 体质、功法、法宝、事件不依赖固定列表
9. **代价平衡** — 强大力量必有代价
10. **Python 概率强制** — 所有判定必须执行 Python 脚本

## 四、技术规格

- **Python 版本**：3.x
- **依赖**：无（仅标准库 `random`、`sys`、`json`）
- **调用方式**：
  - CLI：`python scripts/calc.py <函数名> [参数]`
  - Import：`from scripts.calc import calc_breakthrough`
- **暗骰机制**：所有结果对玩家仅展示成功/失败，不展示概率值

## 五、使用说明

### GM 运行流程

1. 激活 SKILL.md 技能（用户说"玩修仙游戏"等）
2. 熟读 [service/game-rules.md](../service/game-rules.md) 全部规则
3. 按序执行角色创建：`roll_origin` → `roll_spirit_root` → `roll_physique` → `generate_physique_detail`(若触发) → `roll_stats`
4. 按照 §24 开场模板构建开场
5. 游戏中每轮：描述处境 → 玩家选择 → `python scripts/calc.py` 判定 → 推进剧情

### 常用命令

```bash
python scripts/calc.py roll_origin                          # 出身
python scripts/calc.py roll_spirit_root                      # 灵根
python scripts/calc.py roll_stats                            # 属性
python scripts/calc.py calc_hit 10 8                         # 命中
python scripts/calc.py calc_alchemy 3 50 灵草 100 法器 地火 2 # 炼丹
python scripts/calc.py calc_breakthrough 0.85 1.0 '{}'       # 突破
python scripts/calc.py calc_tribulation_trigger              # 天劫触发
python scripts/calc.py calc_encounter 50                     # 机缘
```

## 六、版本边界与限制

### 6.1 适用范围

- **世界观**：《凡人修仙传·人界篇》（炼气期 → 化神期）
- **不覆盖**：仙界篇、灵界篇、其他修仙世界观

### 6.2 已知限制

- 角色数据由 GM 在对话中手动维护，无持久化存储
- 体质冲突与融合（§2.2.5）规则已定义但无专用判定函数
- 体质二次觉醒（传说级以上）需 GM 自行判决
