# 凡人修仙传-人界篇 · 版本说明 v1.1

> 发布日期：2026-07-23
> 性质：重大架构升级 — AI创作自由度最大化
> 核心主题：从"固定菜单式游戏"转型为"AI动态叙事引擎"

---

## 一、升级概述

v1.0 的游戏开局存在大量固定内容：固定宗门列表、固定丹药名称、固定秘境类型、固定开场模板流程……玩家从菜单中选选项，AI 只是"填空机"。

v1.1 **彻底翻转了这一设计哲学**：所有固定列表被标记为"参考方向"或彻底删除，游戏核心从"菜单驱动"变为"叙事驱动"。AI（GM）现在是真正的创造者——宗门名字、丹药效果、势力关系、开场路径，全部由 AI 根据角色出身、灵根、属性实时生成。

---

## 二、设计原则变更

| 原则 | v1.0 | v1.1 |
|:-----|:-----|:-----|
| 宗门/势力 | 固定 6 个正道宗门列表 | AI 根据出身+灵根+地图动态创造 |
| 丹药/法器 | 固定 8 种丹药名 + 属性列表 | 方向性参考，AI 创造具体丹方和效果 |
| 修仙路径 | 菜单选择（正道/邪修/散修 × 13条） | 叙事自然引出，玩家通过行动选择路径 |
| 事件/秘境 | 固定类型表 + 代表名称 | 方向参考，AI 自由创造具体内容 |
| 符箓/阵法 | 固定类型+代表阵法名 | 方向参考，AI 自由命名和设计效果 |
| 体质/出身 | 概率引擎抽取，描述较固定 | 数值种子保留，叙事完全由 AI 发挥 |
| 拍卖/灵田 | 固定入场费/固定等级对应 | 数值模糊化，AI 根据剧情动态设定 |

### 新增核心原则（§1 第 11 条）

```
AI自主创造：本系统不设任何固定菜单选项。
所有宗门、势力、功法、丹药、事件、NPC等，
均由AI根据角色背景和世界情境动态创造。
参考列表仅供"灵感种子"，不得向玩家展示为固定选项。
玩家的每一个选择应来源于叙事，而非菜单。
```

## 四、保留不变的数值框架

以下为核心游戏机制（概率引擎必需），保持不变：

- 境界体系（修为需求/寿元/突破率）
- 概率公式（命中率 75%、暴击 5%、天劫触发 30% 等）
- 五行相克系数（×1.5 / ×0.7）
- 装备槽位限制
- 属性初始值基准
- 灵根加权抽取权重
- 突破失败分段概率（轻伤 40% / 重伤 25% / 修为尽失 20% / 走火入魔 10% / 死亡 5%）

---

## 五、设计哲学

```
v1.0:  Python 计算概率 → AI 填空式叙事 → 玩家从菜单选择
v1.1:  Python 计算概率 → AI 自由创造叙事 → 玩家从故事中选择行动
```

核心边界：
- **概率引擎**（Python）：保留精确数值 → 保证游戏系统可运转
- **叙事层**（AI/GM）：全部释放 → 保证每次开局独一无二
