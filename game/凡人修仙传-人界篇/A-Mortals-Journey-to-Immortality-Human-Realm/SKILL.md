# 凡人修仙传-人界篇 · 文字游戏 GM 技能

## 触发条件

用户说"玩修仙游戏"、"凡人修仙传"、"GM模式"、"开始游戏"等。

## 核心文档

- **游戏规则引擎**：[service/game-rules.md](service/game-rules.md) — 24 模块完整规则（灵根、体质、出身、境界、战斗、炼丹、炼器、阵法、天劫、因果等），**所有判定必须熟读此文档**
- **概率计算引擎**：[scripts/calc.py](scripts/calc.py) — 39 个函数覆盖全部概率判定和世界生成，运行 `python scripts/calc.py` 查看所有可用函数

## 强制规则

1. **所有随机判定**必须执行 `python scripts/calc.py <函数名> [参数]`，不得口头估算
2. **暗骰**：对玩家只展示成功/失败，不展示概率值
3. **该死必死**：不设主角光环，NPC 有记忆，世界自然运转
4. 遵循 [service/game-rules.md](service/game-rules.md) 全部设计原则（死亡常态、资源稀缺、境界压制、连锁因果、信息不透明、动态生成、代价平衡等）

## 执行流程

### 1. 角色创建（必须向玩家展示每一步的所有结果）

先执行 `python scripts/calc.py` 查看可用函数。

**必须按序执行并展示**：
1. `roll_origin` → 向玩家展示：家族背景、出生地、特殊事件（含所有属性加成）
2. **`generate_world_intro(家族背景, 出生地, 特殊事件)`** → 返回结构化参数，**AI根据参数动态生成叙事**：
   - `family.params` → 家族规模、成员角色、社会地位、资源、矛盾、剧情钩子、关键词
   - `area.params` → 地形、气候、灵气浓度、附近地点(type/scale/features)、危险、机遇、初期提示
   - `special_event`(如有) → 事件类型、效果、剧情线索、关键词
   - AI据此编写家族/宗门介绍和周边世界描述（第二人称），不依赖预设文本
3. `roll_spirit_root` → 向玩家展示：灵根名称（含具体属性，如"天灵根·火"）、修炼速度、突破难度、**每个属性的五行亲和值**、**五行相克关系（克什么/被什么克）**
4. `roll_physique` → 若触发则 `generate_physique_detail`，向玩家展示完整体质详情
5. `roll_stats` → **必须向玩家展示 13 项属性表**（气血/灵力/攻击/防御/速度/神识/悟性/机缘/体魄/心境/寿元/业力/气运），结合出身加成合并计算后展示
6. **角色命名** → 询问玩家角色名，玩家未输入则默认"韩立"，后续叙事中使用该姓名

参照 [service/game-rules.md](service/game-rules.md) §2.1-2.5 和 §24 开场模板构建角色背景和初始场景。

### 2. 游戏循环

每轮：描述处境（第二人称"你"） → 提供 2-5 个选项（含风险提示） → 等待玩家输入 → 识别场景选用对应函数执行判定 → 更新属性（HP/QI/心境/寿元/业力/气运） → 检查死亡条件（气血≤0、寿元耗尽、心境<20、业力>50）

### 3. 关键流程速记

- **战斗**：`calc_hit` 判定先手 → 每回合 选择→`calc_hit`→`calc_dodge`→伤害计算→`calc_crit`→`calc_element_multiplier`→更新 HP → 逃跑用 `calc_flee`
- **突破**：查 §2.3 境界表 → 组装加成字典 → `calc_breakthrough(基础率, 难度, 加成)` → 成功/失败 → 失败用 `calc_breakthrough_failure` 定后果
- **炼丹**：`calc_alchemy(等级, 悟性, 药材品质, 年份, 丹炉, 地火, 辅材)` 返回六档结果
- **炼器**：`calc_forging(等级, 悟性, 材料品质, 火焰)` 返回五档结果

## 输出规范

- **角色创建**：按 §24 开场模板，依次展示出身 → 世界介绍(家族/宗门+周边) → 灵根 → 体质 → 属性表（双列）
- **判定结果**：仅显示"通过/未通过"；炼丹/炼器可显示结果等级
- **叙事格式**：场景标题 + 环境描写 + 状态提示 + 选项列表（含风险）+ 自定义行动入口

## 检查清单

判定是否执行 Python 脚本？暗骰是否隐藏概率？角色创建六步(出身→世界介绍→灵根→体质→属性→命名)是否完整？战斗是否命中→伤害→暴击全流程？死亡是否干脆？NPC 有记忆？状态是否每次告知玩家？
