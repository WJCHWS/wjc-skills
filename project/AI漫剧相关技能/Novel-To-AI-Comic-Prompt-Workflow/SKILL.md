# 小说改编AI漫剧提示词生成

## 目标

将输入文件夹下的小说内容（用户指定或默认的`小说原文文件夹`），按照模板生成漫剧制作所需的提示词文件，每章对应一个提示词文件。

## 触发使用技能

- 用户提到"小说改编漫剧"
- 用户提到"生成漫剧提示词"
- 用户指定文件夹并要求生成提示词

## 输入参数

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| source_dir | 字符串 | 否 | 小说原文文件夹 | 小说章节文件所在的文件夹路径 |
| output_dir | 字符串 | 否 | 生成提示词文件夹 | 提示词文件输出的目标文件夹路径 |

**参数说明**：
- 用户可以指定输入文件夹和输出文件夹路径
- 如果用户未指定，将使用默认值
- 默认文件夹名：`小说原文文件夹`（读取）、`生成提示词文件夹`（输出）

## 适用边界

- 输入文件夹（用户指定或默认的`小说原文文件夹`）包含小说章节文件（每个文件一章）
- 需要根据template下的模板生成提示词
- 输出格式为md文件

## 强制要求

- 必须先读取template下的模板文件，仅提取分镜脚本、角色图片生成、角色三视图、场景图片生成4部分，跳过小说内容改写
- 必须遍历输入文件夹（用户指定或默认的`小说原文文件夹`）下所有小说章节文件
- 每章生成一个对应的提示词文件，包含4部分内容（角色图片生成、角色三视图通用提示词、场景图片生成、分镜脚本）
- 每个提示词文件开头必须包含导航目录
- 每个角色用三级标题独立分割，每个场景用三级标题独立分割

## 执行流程

1. **读取模板文件**：读取 `template/小说改编AI漫剧简易流程提示词.md`，从模板中提取分镜脚本、角色图片生成、角色三视图、场景图片生成4部分，跳过小说内容改写部分
2. **确定输入文件夹**：判断用户是否提供了 `source_dir`，若提供则使用用户指定的文件夹；若未提供，则默认读取 `小说原文文件夹`
3. **扫描小说文件夹**：读取确定的输入文件夹下所有小说章节文件，提取章节列表
4. **全局角色/场景分析**：先扫描全部章节，提取全局角色列表和场景列表，确保跨章节一致性
5. **确定输出文件夹**：判断用户是否提供了 `output_dir`，若提供则使用用户指定的文件夹；若未提供，则默认输出到 `生成提示词文件夹`
6. **逐章生成提示词**：
   - 读取当前章节内容
   - 按照模板结构生成提示词（包含分镜脚本、角色图片生成、场景图片生成）
   - 角色三视图通用提示词直接引用模板原版，不需要根据章节定制
   - 在角色和场景生成步骤中，参考全局角色/场景设定，保持一致性
   - 文档开头生成导航目录，每个角色和场景用独立子标题分割
7. **输出文件**：将生成的提示词文件保存到确定的输出文件夹下

## 输出规范

### 文件命名规则

- 输出目录：`{输出文件夹}/`（输出文件夹为用户指定的 `output_dir` 或默认的 `生成提示词文件夹`）
- 文件名：与原章节文件名对应，后缀为 `.md`
- 示例：原文件 `第一章.txt` → 输出 `第一章_提示词.md`

### 文件内容结构

每个提示词文件应包含以下4部分，文档开头必须包含导航目录：

#### 导航目录

```
## 目录

1. [角色图片生成](#一角色图片生成)
   - [角色1](#角色1)
   - [角色2](#角色2)
   - ...
2. [角色三视图通用提示词](#二角色三视图通用提示词)
3. [场景图片生成](#三场景图片生成)
   - [场景1](#场景1)
   - [场景2](#场景2)
   - ...
4. [分镜脚本](#四分镜脚本)
```

#### 各部分内容要求

1. **角色图片生成**
   - 分析当前章节出现的角色
   - 参考全局角色设定，保持风格统一
   - **每个角色用三级标题独立分割**，格式：`### 角色名`
   - 按照模板格式输出角色生图提示词

2. **角色三视图通用提示词**
   - **直接引用模板中的原版提示词**，不需要根据章节角色定制
   - 保持模板中的排版布局示意和比例要求

3. **场景图片生成**
   - 分析当前章节出现的场景
   - 参考全局场景设定，保持风格统一
   - **每个场景用三级标题独立分割**，格式：`### 场景名`
   - 按照模板格式输出场景生图提示词

4. **分镜脚本**
   - 基于当前章节内容，生成分镜脚本提示词
   - 要求：10秒为一段，最少3个分镜，统一风格和色调

### 跨章节一致性处理

- 角色设定：在多个章节出现的同一角色，必须保持相同的特征描述
- 场景设定：在多个章节出现的同一场景，必须保持相同的风格和基调
- 风格统一：整个小说的漫剧风格必须保持一致（如3D/2D、色彩基调等）

## 检查

- 是否读取了所有小说章节文件？
- 是否为每章生成了对应的提示词文件？
- 输出文件是否保存在正确的目录？
- 角色和场景是否在各章节间保持一致性？
- 提示词文件是否包含完整的4个部分（角色图片生成、角色三视图通用提示词、场景图片生成、分镜脚本）？
- 文档开头是否包含导航目录？
- 每个角色是否用三级标题独立分割？
- 每个场景是否用三级标题独立分割？
- 是否使用了正确的文件命名规则？
- 当用户未指定输入文件夹时，是否正确使用了默认的`小说原文文件夹`？
- 当用户未指定输出文件夹时，是否正确使用了默认的`生成提示词文件夹`？
